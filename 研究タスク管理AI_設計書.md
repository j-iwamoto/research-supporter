## 1. プロジェクト概要

### 1.1 背景と課題

修士研究を進める中で、以下の課題が存在する：

- 日々の研究活動の記録が残らず、週報作成時に「今週何をやったか」を思い出すのが困難
- 研究中に思いついたアイデアが散逸し、後から参照できない
- タスクの優先順位が曖昧になり、研究の方向性を見失いがち
- 進捗の可視化ができておらず、モチベーション維持が難しい

### 1.2 プロジェクトの目的

本プロジェクトでは、AI（LLM）を活用した研究タスク管理ツールを開発し、以下を実現する：

1. 最小限の入力で日々の研究活動を記録
2. 記録を自動で分類・整理し、週報ドラフトを自動生成
3. アイデアを蓄積・整理し、関連するアイデア同士を紐付け
4. ダッシュボードで研究の進捗を可視化

### 1.3 ターゲットユーザー

- 大学院生（修士・博士）
- 研究活動を行う学部生
- 個人で研究プロジェクトを進める社会人研究者

### 1.4 技術スタック

| レイヤー | 技術 | 選定理由 |
|---------|------|----------|
| フロントエンド | React + TypeScript + Vite | コンポーネント指向、型安全性、高速な開発体験 |
| UIライブラリ | Tailwind CSS + shadcn/ui | 高速なスタイリング、一貫したデザイン |
| バックエンド | Python + FastAPI | Python習熟度、高速なAPI開発、型ヒントサポート |
| データベース | Cloud Firestore | スキーマレス、リアルタイム同期、GCP統合 |
| AI/LLM | Gemini API (gemini-1.5-pro) | 日本語性能、GCP統合、コスト効率 |
| インフラ | Cloud Run | サーバーレス、オートスケール、コンテナベース |
| 認証 | Firebase Authentication | Google認証、セッション管理 |

## 2. 機能要件

### 2.1 日報記録機能

**概要：** 自然言語で研究活動を記録し、AIが自動で分類・構造化する

#### 入力方法
- テキスト入力（チャット形式）
- 音声入力（将来拡張）

#### AIによる自動処理
- カテゴリ自動分類（実験、論文読み、コーディング、ミーティング、その他）
- タグ自動付与（キーワード抽出）
- 作業時間の推定（オプション）

#### 記録項目

| 項目 | 型 | 必須 | 説明 |
|------|-----|------|------|
| id | string | ○ | 一意識別子（UUID） |
| content | string | ○ | 記録内容（自然言語） |
| category | string | ○ | AIが分類したカテゴリ |
| tags | string[] | - | AIが抽出したタグ |
| created_at | timestamp | ○ | 記録日時 |
| user_id | string | ○ | ユーザーID |

### 2.2 週報自動生成機能

**概要：** 1週間分の日報を集約し、週報フォーマットに沿ったドラフトを自動生成する

#### 出力フォーマット
- **今週やったこと：** カテゴリ別にグルーピングして要約
- **来週やること：** 今週の記録から次のアクションを提案

#### 生成トリガー
- 手動：「週報を作成」ボタンをクリック
- 自動：毎週金曜日18:00に下書きを生成（オプション）

### 2.3 アイデアメモ機能

**概要：** 研究中に思いついたアイデアを記録し、AIが整理・関連付けを行う

#### 機能詳細
- アイデアの記録（タイトル + 詳細）
- AIによるタグ自動付与
- 関連アイデアの自動提案（類似度計算）
- アイデアのステータス管理（未着手、検討中、採用、却下）

#### データ項目

| 項目 | 型 | 必須 | 説明 |
|------|-----|------|------|
| id | string | ○ | 一意識別子（UUID） |
| title | string | ○ | アイデアのタイトル |
| description | string | - | 詳細説明 |
| tags | string[] | - | AIが付与したタグ |
| status | enum | ○ | 未着手/検討中/採用/却下 |
| related_ideas | string[] | - | 関連アイデアのID |
| created_at | timestamp | ○ | 作成日時 |
| user_id | string | ○ | ユーザーID |

### 2.4 ダッシュボード機能

**概要：** 研究の進捗を可視化し、モチベーション維持と振り返りを支援する

#### 表示項目
- 今週の活動サマリー（カテゴリ別の時間・件数）
- 週次推移グラフ（過去4週間の活動量）
- アイデアストック数とステータス分布
- 直近の記録一覧
- AIからの提案（「最近論文読みが少ないですね」等）

### 2.5 機能優先度

| 機能 | 優先度 | MVP | 理由 |
|------|--------|-----|------|
| 日報記録 | 高 | ○ | コア機能、これがないと始まらない |
| 週報自動生成 | 高 | ○ | ユーザーの最大の課題を解決 |
| アイデアメモ | 中 | ○ | 日報と同じUIで実装可能 |
| ダッシュボード（基本） | 中 | ○ | シンプルな集計から開始 |
| ダッシュボード（詳細） | 低 | - | MVP後に拡張 |
| 音声入力 | 低 | - | 将来拡張 |

## 4. データベース設計

### 4.1 コレクション構成

```
firestore/
├── users/
│   └── {userId}/
│       ├── profile: { email, displayName, createdAt }
│       └── settings: { weeklyReportDay, timezone }
│
├── logs/
│   └── {logId}/
│       ├── userId: string
│       ├── content: string
│       ├── category: string
│       ├── tags: string[]
│       ├── createdAt: timestamp
│       └── weekOf: string (YYYY-Www形式)
│
├── ideas/
│   └── {ideaId}/
│       ├── userId: string
│       ├── title: string
│       ├── description: string
│       ├── tags: string[]
│       ├── status: string
│       ├── relatedIdeas: string[]
│       ├── createdAt: timestamp
│       └── updatedAt: timestamp
│
└── weeklyReports/
    └── {reportId}/
        ├── userId: string
        ├── weekOf: string (YYYY-Www形式)
        ├── thisWeek: string
        ├── nextWeek: string
        ├── generatedAt: timestamp
        └── editedAt: timestamp
```

### 4.2 インデックス設計

| コレクション | フィールド | 順序 | 用途 |
|-------------|-----------|------|------|
| logs | userId, createdAt | ASC, DESC | ユーザーの日報を新しい順に取得 |
| logs | userId, weekOf | ASC, ASC | 週報生成時に1週間分を取得 |
| ideas | userId, status | ASC, ASC | ステータス別のアイデア一覧 |
| ideas | userId, createdAt | ASC, DESC | アイデアを新しい順に取得 |
| weeklyReports | userId, weekOf | ASC, DESC | 週報一覧を取得 |

### 4.3 セキュリティルール

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // ユーザーは自分のデータのみアクセス可能
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    
    match /logs/{logId} {
      allow read, write: if request.auth != null 
        && request.auth.uid == resource.data.userId;
      allow create: if request.auth != null 
        && request.auth.uid == request.resource.data.userId;
    }
    
    match /ideas/{ideaId} {
      allow read, write: if request.auth != null 
        && request.auth.uid == resource.data.userId;
      allow create: if request.auth != null 
        && request.auth.uid == request.resource.data.userId;
    }
    
    match /weeklyReports/{reportId} {
      allow read, write: if request.auth != null 
        && request.auth.uid == resource.data.userId;
    }
  }
}
```

## 5. API設計

### 5.1 認証

Firebase Authenticationを使用し、JWTトークンをAuthorizationヘッダーで送信する：

```
Authorization: Bearer <Firebase ID Token>
```

### 5.2 日報API

#### POST /api/logs - 日報作成

**リクエスト：**
```json
{
  "content": "新しいデータセットでモデルを学習した。精度は70%くらい。"
}
```

**レスポンス：**
```json
{
  "id": "abc123",
  "content": "新しいデータセットでモデルを学習した。精度は70%くらい。",
  "category": "実験",
  "tags": ["モデル学習", "精度評価"],
  "createdAt": "2024-01-15T10:30:00Z",
  "weekOf": "2024-W03"
}
```

#### GET /api/logs - 日報一覧取得

**クエリパラメータ：**
- `week`: 週指定（例: 2024-W03）
- `category`: カテゴリでフィルタ
- `limit`: 取得件数（デフォルト: 50）

**レスポンス：**
```json
{
  "logs": [
    {
      "id": "abc123",
      "content": "新しいデータセットでモデルを学習した。精度は70%くらい。",
      "category": "実験",
      "tags": ["モデル学習", "精度評価"],
      "createdAt": "2024-01-15T10:30:00Z"
    }
  ],
  "total": 1
}
```

#### DELETE /api/logs/{id} - 日報削除

### 5.3 週報API

#### POST /api/weekly/generate - 週報生成

**リクエスト：**
```json
{
  "weekOf": "2024-W03"
}
```

```json
{
  "id": "weekly123",
  "weekOf": "2024-W03",
  "thisWeek": "【実験】\n- 新データセットでモデル学習（精度65%→70%に改善）\n- ハイパーパラメータ調整の実施\n\n【論文】\n- Attention機構に関する論文を3本読了",

  "generatedAt": "2024-01-19T18:00:00Z"
}
```

#### PUT /api/weekly/{id} - 週報編集

**リクエスト：**
```json
{
  "thisWeek": "（編集後の内容）",
  "nextWeek": "（編集後の内容）"
}
```

#### GET /api/weekly - 週報一覧取得

### 7.4 必要なGCPセットアップ手順

```bash
# 1. プロジェクト作成
gcloud projects create research-task-ai --name="Research Task AI"
gcloud config set project research-task-ai

# 2. 必要なAPIを有効化
gcloud services enable \
  run.googleapis.com \
  firestore.googleapis.com \
  cloudbuild.googleapis.com \
  artifactregistry.googleapis.com \
  secretmanager.googleapis.com

# 3. Firestoreデータベース作成
gcloud firestore databases create --location=asia-northeast1

# 4. Artifact Registryリポジトリ作成
gcloud artifacts repositories create research-task-ai \
  --repository-format=docker \
  --location=asia-northeast1

# 5. Secret ManagerにGemini APIキーを保存
echo -n "YOUR_GEMINI_API_KEY" | gcloud secrets create gemini-api-key --data-file=-

# 6. Cloud Build用サービスアカウントに権限付与
PROJECT_NUMBER=$(gcloud projects describe research-task-ai --format="value(projectNumber)")
gcloud projects add-iam-policy-binding research-task-ai \
  --member="serviceAccount:${PROJECT_NUMBER}@cloudbuild.gserviceaccount.com" \
  --role="roles/run.admin"
```

## 8. Gemini APIプロンプト設計

### 8.1 日報分類プロンプト

```
あなたは研究活動を分類するアシスタントです。

以下の研究活動の記録を読み、適切なカテゴリとタグを付与してください。

【カテゴリ】（1つ選択）
- 実験: モデルの学習、評価、データ処理など
- 論文読み: 論文の読解、サーベイ
- コーディング: プログラムの実装、デバッグ
- ミーティング: 指導教員との面談、ゼミ、打ち合わせ
- 執筆: 論文やレポートの執筆
- その他: 上記に当てはまらないもの

【タグ】
- 記録から重要なキーワードを2〜4個抽出
- 専門用語、手法名、ツール名などを優先

【入力】
{content}

【出力形式】
JSON形式で出力してください：
{
  "category": "カテゴリ名",
  "tags": ["タグ1", "タグ2", ...]
}
```

### 8.2 週報生成プロンプト

```
あなたは研究週報を作成するアシスタントです。

以下は今週の研究活動記録です。これを週報形式にまとめてください。

【週報フォーマット】
1. 今週やったこと
   - カテゴリごとにグルーピング
   - 箇条書きで簡潔に
   - 具体的な成果や数値があれば含める

2. 来週やること
   - 今週の記録から自然に導かれる次のアクションを提案
   - 3〜5項目程度

【今週の記録】
{logs}

【出力形式】
Markdown形式で出力してください。
```

### 8.3 アイデア関連付けプロンプト

```
あなたは研究アイデアを整理するアシスタントです。

新しいアイデアと既存のアイデアリストを比較し、関連するアイデアを特定してください。

【新しいアイデア】
タイトル: {new_idea_title}
説明: {new_idea_description}

【既存のアイデアリスト】
{existing_ideas}

【タスク】
1. 新しいアイデアに適切なタグを2〜4個付与
2. 既存のアイデアの中から関連性の高いものを最大3つ選択
3. 関連性の理由を簡潔に説明

【出力形式】
{
  "tags": ["タグ1", "タグ2"],
  "relatedIdeas": [
    {"id": "idea_id", "reason": "関連理由"}
  ]
}
```

### 8.4 AIサジェスト生成プロンプト

```
あなたは研究活動をサポートするアドバイザーです。

以下の研究活動データを分析し、有益なアドバイスを1つ提供してください。

【今週の活動サマリー】
{weekly_summary}

【過去4週間のトレンド】
{trend_data}

【アイデアのストック状況】
{ideas_summary}

【アドバイスの観点】
- 活動のバランス（実験、論文読み、執筆などの偏り）
- 継続性（記録頻度の変化）
- アイデアの活用状況
- 具体的で実行可能な提案

【出力】
1〜2文の簡潔なアドバイスを日本語で出力してください。
親しみやすいトーンで、プレッシャーを与えないように。
```
